# SPRG ゲーム形式・UX設計書

## 📋 ゲーム基本構造（決定済み）

### キャラクター体制
- **プレイヤー陣営**: 王様 1体 + 騎士 2体 = 3ユニット
- **敵陣営**: 王様 1体 + 騎士 2体 = 3ユニット
- **合計**: 最大6ユニットがマップに配置される戦術マップ

### ゲーム勝敗判定
```
勝利条件: 敵の王様を撃破
敗北条件: 自軍の王様が死亡
```
- 騎士の死亡は敗北ではない（その後の戦術に影響するが継続可能）
- **永続死亡**: 一度死亡したユニットは復活しない

### マップ環境（タイル移動コスト）
| タイプ | 移動コスト | 特性 |
|--------|----------|------|
| 平原 (plains) | 0 | 最も移動しやすい |
| 山地 (mountain) | 2 | 移動困難、防御に有利 |
| 森 (forest) | 1 | 中程度 |
| 砂漠 (desert) | 0 | 平原と同じ |
| 川 (river) | 1 | 中程度 |

**戦術性**: 高コストタイルはボトルネックとなり、移動経路の選択が重要になる。

## 🎮 ゲームシステム（設計案）

### ステータスシステム（確定）
```
基本ステータス 5種類:
- vitality（活力）: HPの基礎値、被ダメージ軽減
- power（力）: 攻撃力、スキル威力
- guard（防御）: ダメージ軽減率
- luck（運）: クリティカル率、命中率、特殊効果発動率
- speed（速度）: ターン順序決定、回避率

各ステータスにバフ/デバフが存在:
- b.vitality, b.power, b.guard, b.luck, b.speed （バフ）
- d.vitality, d.power, d.guard, d.luck, d.speed （デバフ）
```

### レベルアップシステム（確定）

**経験値計算**:
```
獲得EXP = (敵ベースEXP × 敵レベル) / キャラクターレベル
```

**クラスチェンジ**:
```
レベル 11 に到達 → クラスチェンジ
（具体的なクラスハイアラングはまだ未定義）
```

**成長曲線の特徴**:
- 低レベルの敵を倒しても経験値が少ない（スケーリング機能）
- 同レベル敵: 標準経験値獲得
- 高レベル敵: 多くの経験値を獲得

## 🎯 推奨ゲームループ

### 1. マップ表示フェーズ
```
┌─ ゲーム開始
├─ マップと全ユニット表示
│  ├─ 自軍3ユニット: 可視
│  ├─ 敵軍3ユニット: 可視（後にFoG導入可能）
│  └─ タイル情報表示: 移動コスト目安
└─ ターン開始
```

**UX提案**:
- カメラ: 動的追跡（選択ユニット中心）またはスクロール可能
- ユニット上にHP/ステータス簡易表示
- 選択時に詳細ステータスパネル表示

### 2. ターン進行フェーズ（未決定・案）

**推奨案: 速度値制ターン順序**
```
毎ラウンド開始時:
1. 全ユニットの速度値を集計
2. 速度が高い順にユニット行動を実行（同値は陣営優先）
3. 全ユニット行動終了 → 次ラウンド

例:
敵騎士A (speed=8) → プレイヤー王様 (speed=6) → 敵王様 (speed=5) 
→ プレイヤー騎士B (speed=4) → プレイヤー騎士C (speed=4) 
→ 敵騎士B (speed=3)
```

**メリット**:
- speedステータスが実際に意味を持つ
- 低速ユニットは囲まれるリスク（戦術性向上）
- 敵AIの行動予測がプレイヤーに明確

### 3. ユニット行動フェーズ（1ユニット当たり）

**行動選択肢**（仮）:
```
1. 移動: タイル移動コスト分だけ移動可能
2. スキル使用: 攻撃スキル / バフスキル / 回復スキル
3. 待機: 何もしない（次ターンへ）
4. 防御: 次ターンまでダメージ軽減
```

**UX提案**:
- キャラクター選択 → 移動可能範囲をハイライト表示
- タイル選択 → 移動コスト計算・パス表示（A*アルゴリズム）
- スキル選択 → 対象選択モード（敵/味方範囲表示）

### 4. ダメージ計算（案）

```
基本ダメージ = (攻撃者.power × スキル係数) - (防御者.guard × 0.5)

最終ダメージ = 基本ダメージ × (1 + バフ補正) × (1 - デバフ減少)

クリティカル判定:
- 発動確率 = (攻撃者.luck + スキル補正) %
- クリティカル時: 最終ダメージ × 1.5

命中判定:
- 命中率 = 100% - (防御者.luck × 2%)
```

## 🔴 次のステップ（実装前に決めるべき）

### 優先度 S（必須）
1. **ダメージ計算式の確定** → power/guard/luck の係数値を決定
2. **バフ・デバフの重複ルール** → 加算か乗算か
3. **騎士の死亡時の挙動** → 敗北条件か、単なる戦術不利か

### 優先度 A（実装前）
4. スキル種別と具体的な効果一覧
5. クラスチェンジによる具体的なステータス変化
6. FOW（視界制限）を実装するか判断

### 優先度 B（後からでも可）
7. パーティクルエフェクトの豪華さレベル
8. サウンド・BGM計画
9. 難易度設定（敵AIの強さ調整）

## 📊 想定プレイフロー

```
1. 初期配置: プレイヤー王様 + 騎士×2 vs 敵王様 + 騎士×2
   マップ: 10×10 グリッド、ランダムタイル

2. ターン1-3: お互いが接近、初期スキル交換
   └─ 小競り合い: 低威力スキルでダメージ蓄積

3. ターン4-6: 本格的な交戦
   ├─ 騎士が前衛として敵を牽制
   ├─ 王様がバフスキルで支援
   └─ 誰かが死亡する可能性

4. ターン7+: 最終決戦
   ├─ 一方の王様が追い詰められる
   └─ 勝敗決定（Lv違いで5-15ターン程度想定）

5. 戦闘後:
   ├─ 生き残ったユニットが経験値獲得
   ├─ Lv11で誰かがクラスチェンジ
   └─ 次のマップへ
```
