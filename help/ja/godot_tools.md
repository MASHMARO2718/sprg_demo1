# Godot Tools（VS Code 拡張）の主な機能と使い方

VS Code 向け拡張機能「Godot Tools」は、Godot プロジェクトの編集・デバッグをエディタ側で強力に支援します。以下に主要な機能と設定手順をまとめます。

---

## 1. GDScript の言語機能（コード編集の支援）

- **構文ハイライト** — コードを種類ごとに色分け表示する
- **オートコンプリート** — 入力中に候補を表示する
- **Ctrl+クリックでジャンプ** — 関数や変数の定義へ移動する
- **ホバー時のスニペット／説明** — 識別子にマウスを載せるとドキュメントや情報を表示する
- **Alt+O で `.gd` と `.tscn` を切り替え** — スクリプトとシーンファイルを素早く行き来する
- **静的型付けのサポート** — 型注釈があると補完の精度が上がりやすい

---

## 2. GDScript デバッガー

Godot エディタを起動した状態で、**VS Code から直接デバッグ**できます。Godot Tools の重要な機能のひとつです。

### 初回セットアップ

1. Godot エディタを起動する
2. VS Code で `F1` を押し、`Run and Debug`（実行とデバッグ）を開く
3. `create a launch.json file`（launch.json の作成）を選ぶ
4. **`Debug Godot`** を選択する
5. 以降は **`F5`** でデバッグを開始できる

### デバッグでできること

- **ブレークポイント** — 指定行で実行を一時停止する
- **ステップ実行** — 1 行ずつ進める（Step Over / Step In / Step Out）
- **変数のウォッチ** — 実行中の変数の値を確認する
- **コールスタック** — 関数呼び出しの履歴を表示する
- **シーンツリー** — 現在アクティブなノード構成を確認する
- **インスペクタ** — ノードや変数などをエディタ側で確認・編集する

---

## 3. Godot エディタと VS Code の連携（外部エディタ）

Godot 上で GDScript をダブルクリックしたときに、自動的に VS Code で開くようにできます。

1. Godot エディタを開く
2. **Editor Settings（エディタ設定）** → **Text Editor（テキストエディタ）** → **External（外部）**
3. **Use External Editor（外部エディタを使用）** にチェックを入れる
4. **Exec Path** に VS Code の実行ファイルのパスを指定する（例: Windows では `Code.exe` の場所）
5. **Exec Flags** に次を入力する: `{project} --goto {file}:{line}:{col}`

これで Godot と VS Code を併用しやすくなります。

---

## 4. GDResource 系ファイルのサポート

- **`.tscn`（シーン）** — 構文ハイライトやシーンプレビューなどの支援
- **`.tres`（リソース）** — 同様の編集支援

---

## 5. VS Code 側の設定（`settings.json`）

Godot 4 用の実行ファイルパスを指定します。

```json
"godotTools.editorPath.godot4": "C:\\path\\to\\Godot.exe"
```

Godot 3 を使う場合は、設定キーをプロジェクトに合わせて **`godot3`** 側に変更します（拡張機能のドキュメントを参照）。

---

## 6. 典型的なワークフロー例

1. Godot エディタでシーンを配置・編集する
2. VS Code で GDScript を記述する
3. **`F5`** でデバッグを開始する
4. ブレークポイントで変数や処理の流れを確認する
5. 必要に応じて Godot のシーンツリーや実行結果を確認する

エディタとデバッガを一体で使うことで、Godot 開発の効率を上げられます。
